It’s awesome. I have a few users that have read-only rights to this repo and only occasionally make changes themselves. In these cases, I can’t provide commit rights to the repo so what are we to do? Patches.</p>  <p>A patch is basically a change set wrapped up into a single tidy file. The patch can be created by one dev and sent to another to be applied to the VCS. SVN, like most VCSs has very good support for patches. This post describes how to create one.</p>  <p>First, you should update your working directory if possible with “SVN Update”:</p>  <p><img alt="image" border="0" height="294" src="http://lh3.ggpht.com/_IKD9WtY5kxU/S5VEGnQ7XSI/AAAAAAAAAqo/Qxqm-DKt618/image%5B2%5D.png?imgmax=800" style="border-right-width: 0px; display: block; float: none; border-top-width: 0px; border-bottom-width: 0px; margin-left: auto; border-left-width: 0px; margin-right: auto;" title="image" width="449" /> </p>  <p>Normally you would go to the Commit screen to apply your changes. Since you don’t have commit access, this won’t work, so instead right-click and go to “TortoiseSVN” &gt; “Create Patch”:</p>  <p><img alt="image" border="0" height="290" src="http://lh5.ggpht.com/_IKD9WtY5kxU/S5VEHOyv1rI/AAAAAAAAAqs/VoaH7wbAaiU/image%5B5%5D.png?imgmax=800" style="border-right-width: 0px; display: block; float: none; border-top-width: 0px; border-bottom-width: 0px; margin-left: auto; border-left-width: 0px; margin-right: auto;" title="image" width="519" /> </p>  <p>A dialog will show you all the changes it has detected; you can double click each file to diff it. Choose the changes you want included in the patch and click “OK”:</p>  <p><img alt="image" border="0" height="392" src="http://lh5.ggpht.com/_IKD9WtY5kxU/S5VEHTa2UbI/AAAAAAAAAqw/viJ1aMvMLhY/image%5B8%5D.png?imgmax=800" style="border-right-width: 0px; display: block; float: none; border-top-width: 0px; border-bottom-width: 0px; margin-left: auto; border-left-width: 0px; margin-right: auto;" title="image" width="417" /> </p>  <p>Save the patch somewhere handy:</p>  <p><img alt="image" border="0" height="469" src="http://lh4.ggpht.com/_IKD9WtY5kxU/S5VEHxUPigI/AAAAAAAAAq0/YcyAkQJVNUs/image%5B11%5D.png?imgmax=800" style="border-right-width: 0px; display: block; float: none; border-top-width: 0px; border-bottom-width: 0px; margin-left: auto; border-left-width: 0px; margin-right: auto;" title="image" width="571" /> </p>  <p>Send the patch file off to your committer and you’re done! Go ahead and open it up in a text editor if you want to see how these work. It’s basically a snippet of each of the pieces of code you changed, all bundled up into a nice text file.</p>  <h3>Applying a Patch to a Subversion Repo</h3>  <p>Of course the process of applying patches is simple, too. Right-click on the patch file and choose “TortoiseSVN” &gt; “Apply Patch”:</p>  <p><img alt="image" border="0" height="402" src="http://lh6.ggpht.com/_IKD9WtY5kxU/S5VEIGxMayI/AAAAAAAAAq4/yPmm9gP1Dog/image%5B14%5D.png?imgmax=800" style="border-right-width: 0px; display: block; float: none; border-top-width: 0px; border-bottom-width: 0px; margin-left: auto; border-left-width: 0px; margin-right: auto;" title="image" width="458" /> </p>  <p>Choose the SVN working directory to which the patch should be applied:</p>  <p><img alt="image" border="0" height="599" src="http://lh3.ggpht.com/_IKD9WtY5kxU/S5VEIw2q5mI/AAAAAAAAAq8/3hLIJoBetJU/image%5B17%5D.png?imgmax=800" style="border-right-width: 0px; display: block; float: none; border-top-width: 0px; border-bottom-width: 0px; margin-left: auto; border-left-width: 0px; margin-right: auto;" title="image" width="719" /> </p>  <p></p>  <p></p>  <p></p>  <p>You’ll see a list of the patched files and have the opportunity to review each change:</p>  <p><img alt="image" border="0" height="183" src="http://lh4.ggpht.com/_IKD9WtY5kxU/S5VEJGTKozI/AAAAAAAAArA/_PepYur6bNo/image%5B20%5D.png?imgmax=800" style="border-right-width: 0px; display: block; float: none; border-top-width: 0px; border-bottom-width: 0px; margin-left: auto; border-left-width: 0px; margin-right: auto;" title="image" width="216" /> </p>  <p>Then right click to apply some or all of the changes into the working directory you chose.</p>  <p>The patch has now been applied to your working directory—now would be a good time to commit it via normal means (right-click &gt;&#160; “SVN Commit”):</p>  <p><img alt="image" border="0" height="278" src="http://lh3.ggpht.com/_IKD9WtY5kxU/S5VEJWXojiI/AAAAAAAAArE/O5rHVH7IOos/image%5B23%5D.png?imgmax=800" style="border-right-width: 0px; display: block; float: none; border-top-width: 0px; border-bottom-width: 0px; margin-left: auto; border-left-width: 0px; margin-right: auto;" title="image" width="320" /> </p>  <p>It might seem a little complicated at first, but after you do it once or twice it’ll click as a convenient and effective way to share change sets.</p>
